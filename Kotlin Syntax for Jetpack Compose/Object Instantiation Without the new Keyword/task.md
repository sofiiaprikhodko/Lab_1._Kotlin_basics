# Створення об'єктів без `new`
Для Java-розробника виклик `new Button()` є чітким сигналом виділення пам'яті під новий об'єкт. У Kotlin розробники мови вирішили, що слово `new` є зайвим візуальним шумом.

Виклик конструктора класу в Kotlin виглядає абсолютно так само, як і виклик звичайної функції: `Button()`. Це робить код більш лаконічним і стирає візуальну різницю між створенням об'єкта та викликом функції-компонента.

**Ваше завдання:**
У коді нижче оголошено звичайний клас `Modifier`. Ваша мета — створити екземпляр цього класу і передати його у функцію `Box()`. Зробіть це **без** використання ключового слова `new`.

# Значення за замовчуванням
У Java, якщо ви хочете надати користувачеві можливість викликати метод із різною кількістю аргументів, вам доводиться писати безліч перевантажених версій цього методу (Method Overloading). Це роздуває кодову базу.

У Kotlin параметри можуть мати значення за замовчуванням. Завдяки цьому одна функція `Icon` може замінити десяток перевантажених методів. Ви просто вказуєте лише ті параметри, які відрізняються від стандартних.

**Ваше завдання:**
Викличте функцію `Icon`. З її чотирьох параметрів вам потрібно передати лише два: `iconId` зі значенням `101` та `contentDescription` зі значенням `"Search"`. Використовуйте іменовані аргументи для `contentDescription`, щоб "перестрибнути" через інші параметри.

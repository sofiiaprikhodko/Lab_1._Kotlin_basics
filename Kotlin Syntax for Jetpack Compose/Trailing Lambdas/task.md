# Винесення лямбди за дужки
Це **найважливіше синтаксичне правило**, яке робить декларативний UI можливим у Kotlin.

У Java лямбда-вираз завжди передається всередині круглих дужок методу: `button.setOnClickListener(v -> { ... })`.
У Kotlin діє правило: **якщо лямбда-вираз є останнім параметром функції, його можна (і наполегливо рекомендується) винести за межі круглих дужок**.

Тобто `Row(modifier, { ... })` перетворюється на `Row(modifier) { ... }`. Це робить код схожим на вбудовані конструкції мови (як `if` чи `while`).

**Ваше завдання:**
Викличте функцію `Row`. Передайте перший аргумент `modifier` (рядок `"Fill"`) всередині круглих дужок. Лямбду `content` винесіть за дужки у фігурних дужках `{}`. Всередині лямбди поверніть рядок `"Child"`.

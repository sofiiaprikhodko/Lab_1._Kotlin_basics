# Функції з великої літери
У класичній Java будь-який код має знаходитися всередині класу. Коли ми створюємо UI-компонент у Java, ми зазвичай створюємо екземпляр класу (наприклад, `TextView text = new TextView(context)`).

У сучасному декларативному UI на Kotlin компоненти — це не об'єкти з життєвим циклом, а **звичайні функції верхнього рівня** (Top-level functions), які просто описують, як має виглядати інтерфейс.

Щоб візуально відрізняти функції, які малюють UI, від звичайних функцій бізнес-логіки, розробники домовилися використовувати конвенцію **PascalCase** (назва з великої літери). Хоча технічно це функція, візуально вона виглядає як оголошення класу, що спрощує читання ієрархії.

**Ваше завдання:**
Створіть функцію верхнього рівня `Text`, яка приймає параметр `text` типу `String`. Функція повинна повертати рядок у форматі `"[Text: <переданий_текст>]"`. Зверніть увагу, що назва функції починається з великої літери.

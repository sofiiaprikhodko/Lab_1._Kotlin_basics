# Іменовані аргументи (Named Arguments)
Коли клас або метод у Java приймає 5-10 параметрів (особливо однакового типу), виклик перетворюється на незрозумілий набір значень: `new Text("Title", 24, "Black", true)`. Щоб вирішити це, у Java часто пишуть громіздкі класи-Builder'и (`TextBuilder().text("Title").size(24)...`).

У Kotlin цю проблему вирішено на рівні мови. Ви можете явно вказувати імена параметрів під час виклику функції: `Text(text = "Title", fontSize = 24)`. Це робить код самодокументованим і повністю усуває потребу в патерні Builder.

**Ваше завдання:**
Функція `MyText` приймає три параметри, два з яких мають значення за замовчуванням. Викличте її, явно вказавши імена переданих аргументів: `text` зі значенням `"Compose"` та `fontSize` зі значенням `24`. Параметр `color` пропустіть.

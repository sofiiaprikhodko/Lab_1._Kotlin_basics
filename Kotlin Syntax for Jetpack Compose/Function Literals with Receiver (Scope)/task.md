# Контекст виконання (Scope)
Це одна з "магічних" особливостей Kotlin. Чому певні модифікатори вирівнювання (наприклад, `align`) можна викликати всередині `Row`, але компілятор забороняє їх викликати всередині `Column`?

Відповідь — **лябмди з одержувачем**. Подивіться на тип аргументу функції: `RowScope.() -> String`. Це означає, що лямбда, яку ви передасте, буде виконуватися так, ніби вона є методом класу `RowScope`. Усі методи цього класу стають доступними всередині фігурних дужок без префіксів. Це дозволяє створювати безпечний та контекстно-залежний API.

**Ваше завдання:**
Викличте функцію `Row`. Всередині фігурних дужок у вас автоматично є доступ до методів `RowScope`. Просто викличте метод `align()` і поверніть його результат.
